    Introduction to the AWS Cloud

    Cloud Computing:

    Cloud computing refers to the on-demand delivery of IT resources and applications via the internet.

    Scalability

    In cloud computing, the term scalability means the ability to resize your resources as necessary. By using AWS, customers can grow, shrink, and adapt their consumption of services to meet seasonal requirements, launch new services or products, or simply accommodate new strategic directions.

    Agility

    Three of the main factors that influence agility are:

    1. Increasing speed.
    2. The case of experimentation
    3. And cultivating a culture of innovation

    AWS facilities are all over the world, you can literally have a global reach with just a moment’s notice.

    Cloud computing allows you to experiment quickly, with low cost and low risk. This allows for more experimentation, more often, which can lead to the discovery of innovations and new configurations.

    Elasticity (Auto Scaling/Elastic Load Balancing)

    Elasticity is the power to scale computing resources up or down easily.

    Reliability (Availability Zones)

    Reliability is the ability of a system to recover from infrastructure or service failures. In cloud computing, reliability means being able to acquire computing resources to meet demand and mitigate disruptions.

    Fault tolerance means a system can remain operational even if some of the components of that system fail.

    Security

    The AWS Cloud provides the ability to monitor your IT resources continually, so that you can spot any changes at once and respond if necessary.

    Ways to access AWS

    AWS users can create and manage resources in three unique ways:

    1. Using the AWS Management Console (Graphical Interface)
    2. The AWS Command Line Interface (AWS CLI, control AWS from the command line)
    3. The AWS software development kits (SDKs, access AWS using a variety of popular programming languages)

    AWS Core Services

    EC2(Elastic Compute Cloud)

    Compute refers to compute, or server resources that are being presented.
    Cloud refers to the fact that these are cloud-hosted compute resources.
    Elastic refers to the fact that if properly configured, you can increase or decrease the amount of servers required by an application automatically according to the current demands on that application.
    Steps to launch new EC2 instance:

    1. Log to the AWS console
    2. Choose a region where we’re going to host our instance
    3. Launch the EC2 wizard, select the AMI (Amazon Machine Image)
    4. Select instance type, referring to the hardware capabilities
    5. Configuration includes network, storage, key pairs

    SSH Connection
    Default user is ec2-user, then copy the public DNS address, formation is ec2-user@yourpublicDNSaddress

    EBS(Amazon Elastic Block Store)

    • EBS volumes can be used as a storage unit for your Amazon EC2 instances, so whenever you think you need disk space for your instances running on AWS, you can think about them.
    • EBS volumes are designed for being durable and available. This means that the data in a volume is automatically replicated across multiple servers running in the Availability Zone.
    • You can resize your EBS without stop the instance.
    • You can have encrypted EBS volume at no additional cost. The encryption occurs on the EC2 side.
    • EBS must have the same Availability Zone with EC2 instance.
