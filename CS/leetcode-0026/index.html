    0026. Remove Duplicates from Sorted Array (link)

    Description

    Given a sorted array nums, remove the duplicates in-place such that each element appear only once and return the new length.
    Do not allocate extra space for another array, you must do this by modifying the input array in-place with O(1) extra memory.

    26.1 Basic method

    Thought:
    Compare the item with the next item in the list. If they are the same, pop the item until the item is not equal to the next item. There is something very tricky that when pop an item from the list or add an item to the list when you in the loop, the length of the list will change and also index of element in the list. So, it is easy to make mistakes when you process a list from left to right. You can try to process the list from right to left.
    从左到右对nums中的元素进行遍历，对比元素和下一个元素是否相等，相等则删除，不等则进入下一个循环。当用for range() len()对列表进行循环的时候，如果在循环过程中，对列表有增加或者删除的操作，会影响后面元素的index，这一点比较容易出错。为了避免这个容易出错的地方，可以选择用reversed()倒着对列表进行遍历，对时间复杂度没有影响。

    Time Complexity: O(n)

    26.2 Fastest method

    Thought: Use the extra variable index to store the number of unique value, and nums[index] is the previous number you want to compare. So, if you find an element that diffirent with the previous value, then put it after the previous value, and index += 1. If you find an element that is the same with previous, then move forward to check next value.
    用一个变量来存储不同值的个数，同时num[index]也代表了判断过的值。如果遍历到的值和num[index]不同，则代表又发现了一个新值，index+1，然后把新值放在index+1之后的位置上。一直到循环结束。
    Time Complexity: O(n)

    26.3 Shortest method

    Similar to 26.1, replace reversed() with range(a,b,-1) to loop a list in reverse.
    和26.1比较像，把reversed()函数用range()自带的参数取代，在第三个参数位置写-1，就能返回一个倒置的迭代器。

    26.4 Summary

    1. When you modify a list (insert or remove items) during iteration, the result maybe incorrect. Since the index of items has already updated after you modify (insert or remove) the list, you may skip some items you want to process. There are two ways I prefer to solve this kind of problem:

      1. Iterate the list in reverse;
      2. Use while loop and don’t increment index when remove an item from list in this problem (like 26.1).

    2. List methods comparison:

      Operation Example Complexity Class In-place or Not Notes
      Index l[i] O(1) N/A
      Store l[i]=0 O(1) N/A
      Length len(l) O(1) N/A
      Append l.append(5) O(1) In-place
      Pop l.pop() O(1) In-place
      Clear l.clear() O(1) In-place
      Slice l[a:b] O(b-a) Not
      Pop l.pop(k) O(k) In-place
      Extend l.extend(...) O(len(…)) In-place Only depends on len of entension
      Construction list(...) O(len(…)) N/A Depends on length of … iterable
      Check ==, != l1 == l2 O(N) N/A
      Insert l[a:b] = ... / l.insert(i,x) O(N) In-place
      Delete del l[i] O(N) In-place Depends on i; O(N) in worst case
      Containment x in/not in l O(N) N/A linearly searches list
      Copy l.copy() O(N) N/A
      Remove l.remove(...) O(N) In-place
      Count l.count(x) O(N) N/A
      Index l.index(x[, start[, end]]) O(N) N/A
      Extreme value min(l)/max(l) O(N) N/A Linearly searches list for value
      Reverse l.reverse() O(N) In-place
      Iteration for i in l: O(N) N/A Worst: no return/break in loop
      Sort l.sort() O(NlogN) In-place
      Multiply k*l O(k N) Not

    3. Ways to remove item from a list:

      1. list.pop([index]): The pop() method removes the item at the given index from the list and returns the removed item. Default is remove the last element.
      2. list.remove(element): The remove() method removes the first matching element (which is passed as an argument) from the list. Return None.
      3. Slice a list to remove item. list[:2] + list[3:].
      4. del list[10]. Return None.
      5. list.clear() will remove all items in a list.
